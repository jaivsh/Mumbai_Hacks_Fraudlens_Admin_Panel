07 Deepfake & synthetic-media APIs (platform)

Dedicated liveness / deepfake / presentation-attack endpoints are part of the FraudLens extended security suite (mobile SDK & partner integrations). They are not served by the Blackbox service above (Blackbox = SMS + live vishing WebSocket).

Illustrative REST shapes for integration specs (base URL issued per deployment / SDK):

MethodExample pathPurpose
POST/v1/integrity/liveness/sessionStart or continue liveness challenge; returns session id + status
POST/v1/integrity/media/analyzeShort-lived upload or signed URL → deepfake / synthetic-media risk score + audit id
POST/v1/integrity/voice/verifyOptional voice anti-spoofing (partner pipeline)

Wire these to the same policy and audit spine as Chronos (decision + hash) when enterprises require evidence packs.
